# Maze-Escape

Maze-Escape is a 2D Godot maze escape game. The player explores dungeon-style maze levels, avoids hostile dwellers, uses safe camp areas when available, and reaches the exit to progress through the level sequence.

## Gameplay

The goal of each level is to guide the player from the starting area to the exit while avoiding dwellers. Entering the `FinishArea` completes the current level. If the active level is not the final level, the success screen advances to the next maze; after the final level, progression resets and returns to the main menu.

Dwellers patrol the maze, detect the player within their detection radius, chase while the player is visible, and search near the last known player position when the player escapes detection. If a dweller gets close enough to the player, the game changes to the game over screen.

## Controls

The player uses Godot's built-in `ui_*` input actions:

- Move up: `ui_up`
- Move down: `ui_down`
- Move left: `ui_left`
- Move right: `ui_right`

On menu/result screens:

- Enter: confirm / continue
- Escape: quit

## Mechanics

- Player movement: `scripts/player.gd` reads directional input every physics frame, normalizes diagonal input so diagonal movement is not faster, and moves the `CharacterBody2D` with `move_and_slide()`.
- Level progression: `scripts/level_manager.gd` stores five level configurations and controls transitions between the menu, level scenes, success screen, and game over screen.
- Finish area: `scripts/finish_area.gd` listens for the player entering the exit area and asks `LevelManager` to complete the current level.
- Dweller detection: dwellers can spot the player through a detection area and distance checks. Detection is disabled while the player is inside a camp.
- Catch condition: if a dweller is within its attack radius, it triggers `scenes/game_over.tscn`.
- Camps: levels with a `Camp` area allow the player to hide. When the player enters camp, dwellers stop chasing/searching and return to patrol. Camp tiles are blocked in the dweller navigation grid so dwellers do not path into the safe zone.
- Day/night cycle: enabled on later levels. `scripts/day_night_cycle.gd` darkens the scene over timed phases and calls each dweller's `set_night_intensity()`. At night, dwellers become faster and detect from farther away.
- Extra dwellers: later levels can spawn additional dwellers using `scripts/dweller_spawner.gd`. Spawn points are chosen from walkable tiles while keeping distance from the player, existing dwellers, and camp.
- Audio: normal background music and chase music are autoloaded. Chase music starts when a dweller enters chase state and stops when the chase ends. Dwellers also play a roar sound near the player.

## Algorithms Used

### Grid-Based Pathfinding with AStarGrid2D

Dweller navigation is built with Godot's `AStarGrid2D` in `scripts/dweller.gd`.

1. The dweller reads the active `TileMap` used rectangle.
2. Each tile with collision polygons is marked as solid.
3. Solid cells are expanded with clearance offsets based on the dweller collision shape so paths do not clip walls.
4. Camp cells are marked solid when a camp exists.
5. Paths are requested from the dweller's current tile to patrol targets, the player position, or the last known player position.

Diagonal movement is disabled with `AStarGrid2D.DIAGONAL_MODE_NEVER`, so dwellers path through the maze in four cardinal directions.

### Finite State Machine Enemy AI

Each dweller uses a finite state machine:

- `PATROL`: follow generated patrol points around reachable maze cells.
- `CHASE`: path to the player's current position and repath periodically.
- `SEARCH`: move around the player's last known position for a limited duration.
- `CAMP_INVESTIGATION`: reserved state for camp-related behavior.

The state machine controls music transitions, path targets, and how dwellers respond when they gain or lose the player.

### Random Reachable Patrol Generation

Patrol points are generated by collecting walkable cells from the maze grid, shuffling them, and keeping candidates that have a valid A* path from the dweller's origin. This gives dwellers varied patrol routes while avoiding unreachable maze sections.

### Collision and Unstuck Handling

Dwellers move along A* paths one grid axis at a time. The movement code probes upcoming motion with `test_move()`, tries an alternate axis when appropriate, and starts a short unstuck movement if the dweller remains blocked near a waypoint. After unstucking, the dweller repaths to its current goal.

### Threaded Scene Loading

The loading screen uses `ResourceLoader.load_threaded_request()` and `ResourceLoader.load_threaded_get_status()` to load the first game scene while updating a progress bar.

## Project Structure

```text
assets/                  Art, UI, sprites, tilesets, and audio assets
scenes/                  Godot scenes for menus, levels, player, dwellers, and result screens
scripts/                 GDScript gameplay, AI, level, loading, and menu logic
project.godot            Godot project configuration and autoload setup
default_bus_layout.tres  Audio bus configuration
```

Important scripts:

- `scripts/player.gd`: player movement.
- `scripts/dweller.gd`: dweller AI, pathfinding, detection, states, movement, audio cues, and catch logic.
- `scripts/dweller_spawner.gd`: extra dweller spawning for advanced levels.
- `scripts/day_night_cycle.gd`: day/night visual cycle and enemy difficulty scaling.
- `scripts/level_manager.gd`: level definitions and progression.
- `scripts/finish_area.gd`: exit trigger.
- `scripts/loadingscreen.gd`: threaded loading screen.
- `scripts/game_success.gd`: next-level success flow.
- `scripts/gameover.gd`: game over flow.

## Requirements

- Godot 4.6 or compatible Godot 4.x version
- Desktop platform supported by Godot

The project is configured with:

- Main scene: `res://scenes/main_menu.tscn`
- Viewport: 1600x900
- Stretch mode: `canvas_items`
- Autoloads: `GameBgMusic`, `ChaseBgMusic`, and `LevelManager`

## Running the Project

1. Open Godot.
2. Import or open this project folder.
3. Run the project from the editor.

The game starts at the main menu. Press the start button to load the first maze.

## Development Notes

- Maze collision is driven by tile collision data in each level's `TileMap`.
- Dwellers expect the level scene to contain `TileMap`, `Player`, and `dweller` nodes.
- Camp support depends on an optional `Camp` `Area2D` node.
- Level completion depends on a `FinishArea` node connected to `finish_area.gd`.
- Later-level difficulty is controlled mostly through data in `LevelManager`, so speed, detection radius, search duration, patrol count, camps, day/night, and extra dwellers can be tuned without changing the core AI code.
